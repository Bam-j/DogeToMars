##현재 문제점##
1. 모바일 환경(ios 사파리 브라우저)에서 핀치 줌의 문제
=> ios가 핀치 줌이나 더블 탭 확대를 html에서 막아도 작동되게 하는 이슈가 있음.
2. 슬라이드 메뉴(랭킹 뷰어용) 시스템 다듬기
3. 클릭 한 번에 1km라고 책정했으나 실제 천체들의 거리를 계산한 결과 무지막지한 수의 클릭 횟수가 요구.
    터치 한 번에 10을 할지 실제 거리를 줄일지에 대한 논의가 필요한 듯
4. 지금 css 꾸미기를 위해 각 div 영역에 테두리 쳐놨음, 꼭 삭제하는 것 잊지말기!

##남은 개발##
1. 코인과 배경이미지 삽입하기 - 자료조사 시급!
2. N회 클릭시 바뀔 배경이미지 로직 설계
3. 서버연동으로 랭킹시스템 구현하기
4. 이미지 크기와 폰트 flexible하게 만들기
5. 로컬스토리지 값 변경이 개발자도구를 통해 너무 쉽게 조작되는 경향이 있어서
    키 값에 대해서 어떤 암호화 과정을 거치고, 해독해서 이용하는 방식으로 바꿔야겠음!

##(팀원들에게) 테스팅 과정에서 로컬스토리지 값 삭제하기##
1. 개발자 도구 > 어플리케이션 > 로컬스토리지 >https://bam-j.github.io에 키와 값이 저장된 값이 있을텐데
이를 우클릭해서 삭제 하면 스토리지 값 삭제됨
+마찬가지로 값 부분에 원하는 값 넣으면 특정한 값으로 조작 가능
2. 모바일 환경도 인터넷 사용 기록에서 쿠키 및 데이터 삭제나 사용 기록을 삭제하면 된다.

##주의 사항##
웹 호스팅으로 생성된 배포 주소에서 보는 결과 화면과 웹스톰 플러그인으로 열은 html 화면 모습이 다릅니다.
테스팅 할 때 실제 배포될 모습인 호스팅 주소나, 로컬 html파일 열기를 통해 결과물 테스트 확인할 것
